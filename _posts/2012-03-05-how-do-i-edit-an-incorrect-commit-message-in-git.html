--- layout: post title: How do I edit an incorrect commit message in Git? tags: - git - Programming status: publish type: post published: true meta: _edit_last: '1' dsq_thread_id: '708589074' --- Today I wrote an incorrect Git commit message and had to fix it before pushing to the repository. Here is how to do it: 
git commit --amend -m "New commit message"
Example: 
git commit -am "Fixed the mtd report to work on the first day of the motnh"
[hotfix-1.27.2.0 4c45371] Fixed the mtd report to work on the first day of the motnh
3 files changed, 38 insertions(+), 18 deletions(-)

git commit --amend -m "Fixed the mtd report to work on the first day of the month"
[hotfix-1.27.2.0 98680c8] Fixed the mtd report to work on the first day of the month
3 files changed, 38 insertions(+), 18 deletions(-)

git push origin hotfix-1.27.2.0
This should only be done if you haven't pushed to the repository yet. Source: http://stackoverflow.com/questions/179123/how-do-i-edit-an-incorrect-commit-message-in-git
